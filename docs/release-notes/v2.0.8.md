## 中文

### Kimi/DeepSeek thinking 在 Codex CLI TUI 不显示的修复

**症状**: 用户反馈 Kimi 的思考内容应该非常长,但接入 Codex CLI 后看到的 reasoning 段几乎是空的。

**根因**(完整源码追踪见 `docs/investigation/kimi-reasoning-truncation.md`):Codex CLI 0.128 在 `tui/src/history_cell.rs:2783 new_reasoning_summary_block` 检测累积 reasoning buffer 是否包含匹配的 `**...**` markdown 标记 —— OpenAI 自家 o1/o3 reasoning 内置 `**Section Header**`,所以走"显示"分支;**Kimi for Coding / DeepSeek thinking 等纯文本 reasoning 流没有 `**`,被整段判为 `transcript_only=true`,主 UI 完全不渲染**(只在 `/transcript` 命令可见)。

实证:直 curl Kimi for Coding 上游录 5.4MB / 22104 帧 / 32655 字符 reasoning,proxy 转换层 byte-perfect 透传(0 字符丢失),问题精准定位在 Codex CLI TUI 渲染策略。

**修复**: `crates/adapters/src/responses/converter.rs::open_reasoning` 在 reasoning 流首次开启时注入一段 `**Thinking**\n\n` prefix(同时 push 到 `reasoning_acc` 和 emit 一条 `reasoning_summary_text.delta` 事件),让 Codex CLI 命中 bold-header 显示分支。修复后 Kimi/DeepSeek 等纯文本 reasoning 完整显示在主 UI。

测试覆盖: 5 条 reasoning 单测 + 2 条 Kimi fixture integration test 已全部更新,workspace 233+ 项绿。

### `extraHeaders` 支持 `{apiKey}` 模板替换

v1.0.3 backend `proxy.py:381` 在合并 `provider.extraHeaders` 时会把头值里的 `{apiKey}` 占位符替换成实际 API 密钥,典型场景是 DeepSeek 这类同时需要 `Authorization: Bearer ...` 和 `x-api-key: ...` 双头鉴权的 provider —— 用户在 UI 里只配一份 apiKey,extraHeaders 写 `{"x-api-key": "{apiKey}"}` 就能复用。

Rust 改造后 `crates/proxy/src/resolver.rs` 直接把 `{apiKey}` 字面量传到上游 → 所有依赖这个模板的 provider 都会拿到错误的鉴权头 → 401。

本版本在 resolver 转 HeaderMap 前加一行 `v.replace("{apiKey}", &provider.api_key)`,行为对齐 Python。新增单测 `extra_headers_substitute_api_key_template` 覆盖。

## English

### `extraHeaders` now supports `{apiKey}` template substitution

v1.0.3's `backend/proxy.py:381` substituted `{apiKey}` in `provider.extraHeaders` values with the real API key — used by providers like DeepSeek that require both `Authorization: Bearer ...` and `x-api-key: ...`. Users configure one `apiKey` in the UI and write `extraHeaders = {"x-api-key": "{apiKey}"}` to reuse it.

The Rust rewrite (`crates/proxy/src/resolver.rs`) was forwarding the literal `{apiKey}` string to upstream, breaking auth for any provider relying on this template.

This release adds the substitution before HeaderMap conversion and a unit test (`extra_headers_substitute_api_key_template`).
