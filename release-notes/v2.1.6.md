# Codex App Transfer v2.1.6

## 中文

### 多家上游协议修复与跨层 silent failure 观测增强

本版本修了 MiniMax 多轮 400、grok_web 多轮历史丢失、cloud_code (Gemini OAuth) 多轮历史 silent loss 三类 bug,并给 sqlite 持久化与 cache 查询路径补齐稳定 `error_id`。

根因都是 silent fallback 遗留:MiniMax 拒 `role=system` 旧代码没转 user;grok_web 多轮 `assistant.tool_calls` 段被 flatten drop 让 `function_call_output` 变 orphan;`session_cache` 入口接 `Option` 让 prod 可降级;`mapper/cloud_code.rs` 走无 cache gemini 入口;sqlite 与 cache miss 用 `eprintln!` 无 stable token。

本次修复:

- MiniMax provider 把 `role=system` 转 `role=user` + `[System]` 前置 marker,长 system 切片双 pass 预算。
- grok_web flatten 把 `assistant.tool_calls` 渲染成独立 `Assistant (tool call)` 段,补 13 条边界。
- grok_web `session_cache` 改 `&ResponseSessionCache` 必填,类型层面禁止 no-cache prod 路径。
- `mapper/cloud_code.rs` 对齐 gemini_native 主路径接 `global_response_session_cache()`,流末写 cache。
- sqlite 与 `core::input` 加 14 个稳定 `error_id`(`SESSIONS_DB_*` / `CORE_INPUT_PREV_ID_*`),`eprintln!` 兜底。

## English

### Multi-provider protocol fixes and cross-layer silent-failure observability

This release fixes three bugs — MiniMax multi-turn 400, grok_web multi-turn history loss, cloud_code (Gemini OAuth) multi-turn silent-loss — and adds stable `error_id` tokens on sqlite and cache lookup paths.

Root cause: silent-fallback leftovers. MiniMax rejects `role=system` but old code did not convert it; grok_web `assistant.tool_calls` was dropped by flatten, orphaning `function_call_output`; `session_cache` accepted `Option` so the prod path could degrade; `mapper/cloud_code.rs` called the non-cache gemini entry; sqlite plus cache-miss used `eprintln!` without stable tokens.

Changes:

- For MiniMax, `role=system` is converted to `role=user` with `[System]` prefix; long-system chunking uses a two-pass budget.
- grok_web flatten renders `assistant.tool_calls` as a standalone `Assistant (tool call)` segment, plus 13 edge cases.
- grok_web `session_cache` tightened to `&ResponseSessionCache`, making the no-cache prod path impossible at the type level.
- `mapper/cloud_code.rs` aligned with gemini_native main path; wires `global_response_session_cache()` and writes cache at end-of-stream.
- sqlite and `core::input` paths gain 14 stable `error_id` tokens (`SESSIONS_DB_*` / `CORE_INPUT_PREV_ID_*`), with `eprintln!` as fallback.
