## 🔥 关键改动:chat 端原生 web_search 工具支持(逐家文档实证)+ 通用 URL citation 入站处理 + MiniMax builtin preset

v2.1.2 围绕 **web_search 全 provider 实施**展开。Codex.app 默认每轮在 tools 数组里发 OpenAI Responses API 标准的 server-side web_search 工具:

```json
{"type":"web_search","external_web_access":true,"search_content_types":["text","image"]}
```

代理之前对这个工具走 `_ => warn_once + drop`(P5 兜底),模型只能退化到用 MCP 工具(如 Node Repl + JS fetch DDG)绕路联网,效率低。本版**逐家 WebFetch 官方文档实证后** per-provider 适配各自上游 chat completions API 真实支持的形态;同时入站响应通用处理 `delta.annotations` URL citation,任何 chat 上游 URL 引用都能透传给 Codex.app UI。**新加 MiniMax builtin preset 卡片**(用户痛点:之前没有 MiniMax 卡片,必须手动配 base_url)。

### 1. chat 端原生 web_search 工具 — 逐家文档实证后实施

#### Xiaomi MiMo

私有 chat 端 `type:"web_search"`(实测 Codex App 默认配置 + MiMo 多模态文档)。1:1 复刻 [`7as0nch/mimo2codex`](https://github.com/7as0nch/mimo2codex) `reqToChat.ts:196-209`,4 字段透传 `user_location` / `max_keyword` / `force_search` / `limit`(全可选)。OpenAI 的 `external_web_access` / `search_content_types` 在 MiMo 无等价,silent drop。

#### Kimi (Moonshot)

WebFetch `https://platform.kimi.ai/docs/guide/use-web-search` 真原文实证:

```json
{"type":"builtin_function","function":{"name":"$web_search"}}
```

> "When using `$web_search` function, you must disable the thinking ability of the model."

代理在出站时**自动注入** `body["thinking"] = {"type":"disabled"}` 顶级字段(Kimi API 强制要求)— UI 提示文案:**"启用 web_search 时模型 thinking 能力会被禁用(Kimi API 限制)。"**。计费 $0.005/搜索独立于 token。**Kimi For Coding + Kimi 月之暗面同形态**。

#### A 配置开关 + B transparent retry 双层防错

- **A 层**:`Provider.request_options.web_search_enabled`(boolean,**默认 false**)。用户必须显式标 true 才发 web_search 工具上去 — 默认关闭防 MiMo Token Plan 套餐没开 Web Search Plugin 时 400
- **B 层**:`forward.rs::is_web_search_upstream_reject` 4xx 路径识别上游拒绝关键字(实测 MiMo "webSearchEnabled is false" / 通用 "web search.*not enabled|disabled")→ 调 `disable_web_search_for(provider.id)` + **transparent retry without web_search** → 客户端只感知到正常 SSE 流(Codex.app 实测对 4xx JSON error body 卡 Thinking,必须 transparent 重试,否则 session 卡死)

实测 MiMo 用户在 plugin 未开场景 log 流:`WARN auto-disabled web_search for provider <id> → INFO web_search retry status 200 → SUCCESS upstream status 200`,无感降级秒出结果。

#### DeepSeek / MiniMax

WebFetch `api-docs.deepseek.com/api/create-chat-completion` 真原文实证 `"Currently, only function is supported"`(无 builtin web_search / 任何 server-side 搜索能力)。MiniMax 三方实证(`platform.minimaxi.com/docs/api-reference/` + `platform.minimax.io` + liteLLM provider docs)同样 chat 端 tools 仅 function;web_search 仅作 Token Plan MCP 工具存在。两家走显式 drop + 专属 warn key(`web_search:not-supported-by-deepseek-api` / `web_search:not-supported-by-minimax-api`),用户在 log 看到精确 warn key 立即知道是 API 本身不支持。

### 2. 通用 `delta.annotations` URL citation 入站处理(跨 provider)

`ChatToResponsesConverter` 加 `active_annotations` 累计 + `handle_annotations_delta` + `translate_annotation`(`summary` → `snippet` 字段重命名,对齐 OpenAI Responses url_citation schema)。1:1 复刻 mimo2codex `streamToSse.ts:156-163, 338-352`。final message item `content[0].annotations` 从写死 `[]` 改为实际累积值。

**任何 chat 上游模型回答里 URL 引用都通过 `response.output_text.annotation.added` event 透传给 Codex.app UI 显示来源链接** —— P5 之前完全丢失,本版后所有 provider 都受益(不只 web_search 用,普通对话引用网页时也用)。

### 3. MiniMax builtin preset 卡片(用户痛点)

之前用户用 MiniMax 必须手动填 base_url(自定义 provider),不像其他 5 家是 builtin preset 直接选。本版新加第 8 条 builtin preset:`id:"minimax"`,`baseUrl:"https://api.minimaxi.com/v1"`,`apiFormat:"openai_chat"`,`default model: "MiniMax-M2.7"`。

## English

### 1. Native chat-endpoint `web_search` tool support — per-provider documentation-verified rollout

Codex App's runtime sends OpenAI Responses-style `{"type":"web_search","external_web_access":true,"search_content_types":["text","image"]}` to the proxy on every turn. Previous releases dropped this entirely, leaving the model to fall back to MCP tools (e.g. Node Repl + `fetch()`) for web search — slow and roundabout. This release maps it to each upstream provider's actual chat-endpoint web_search form, after WebFetch'ing official documentation per provider:

- **Xiaomi MiMo**: private `type:"web_search"` chat tool with 4 field passthrough (`user_location` / `max_keyword` / `force_search` / `limit`). 1:1 borrow from `7as0nch/mimo2codex` `reqToChat.ts:196-209`
- **Kimi (Moonshot, both For-Coding and Moonshot Platform)**: builtin_function `$web_search` per `platform.kimi.ai/docs/guide/use-web-search`; proxy auto-injects `thinking:{type:"disabled"}` top-level field (mandated by Kimi docs — UI needs note: "enabling web_search disables model thinking — Kimi API constraint")
- **DeepSeek / MiniMax**: documented as not supporting native web_search on chat completions. Explicit drop with dedicated warn key (`web_search:not-supported-by-{deepseek,minimax}-api`) so users can debug from log

#### A + B two-layer fail-safe

- **A**: `Provider.request_options.web_search_enabled` (boolean, **default false**). Users must opt in explicitly — protects MiMo Token Plan customers who haven't activated the Web Search Plugin
- **B**: 4xx upstream rejection (e.g. MiMo `webSearchEnabled is false`) triggers in-process disable cache + **transparent retry without web_search** — the client only sees a normal SSE stream (Codex App, when faced with 4xx JSON, hangs in Thinking; transparent retry is required for usability)

### 2. Cross-provider `delta.annotations` URL citation inbound handling

Any upstream returning citations now flows through `response.output_text.annotation.added` events, with `summary` field renamed to `snippet` to match OpenAI Responses url_citation schema. Borrows from mimo2codex `streamToSse.ts:156-163, 338-352`. Final message item `content[0].annotations` populated with accumulated citations (was hard-coded `[]` before this fix).

### 3. MiniMax builtin preset (user pain point)

Users previously had to manually configure base_url to use MiniMax. This release adds MiniMax as the 8th builtin preset (`id:"minimax"`, `baseUrl:"https://api.minimaxi.com/v1"`, `apiFormat:"openai_chat"`, default `MiniMax-M2.7`) plus the official branded favicon (pink-orange gradient with sound-wave M logo, fetched from `www.minimaxi.com/favicon.ico`).
