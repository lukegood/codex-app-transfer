## 🔥 关键改动:Responses 协议 direct 透传(自定义第三方)+ 诊断闭环 + tracing 全局桥接

v2.1.3 主线是**自定义第三方 + Responses 协议 → Codex.app 直连上游不经代理**(借鉴 codex-account-switch 纯配置写入),配套整套测速 / 上游错误体 / `tracing` → `proxy_telemetry().logs` 桥接的诊断闭环。

### 1. 自定义第三方 + Responses 协议 direct 透传(主线)

之前所有 provider 都强制走本地代理(`127.0.0.1:18080`)做 Responses↔Chat 协议转换。本版让用户在「**自定义第三方**」preset 显式选 `apiFormat=responses` + 填齐 baseUrl + apiKey 时:

- `~/.codex/config.toml::openai_base_url` 直接写**用户填的 baseUrl**
- `~/.codex/auth.json` 直接写**用户填的 apiKey**(不是 gateway key)
- `requires_proxy=false` → 本地 18080 端口**根本不启**
- 适用 OpenAI 官方 / 任何原生实现 OpenAI Responses API 的反代

**v1.x MiMo Token Plan 404 教训保留**:healing 启动时把 8 条 builtin preset 的 `apiFormat` 强制覆盖回 `openai_chat`,bypass 只命中**显式自定义 + 用户主动选透传**的第三方 — informed choice 路径。

backend 配套 `ResponsesPassthroughAdapter`(`crates/adapters/src/passthrough.rs`):字节级透传,SSE envelope / `sequence_number` / resp_id 全由上游产生代理不重写。**例外**:`/responses/compact` 是本仓库私有扩展(OpenAI 上游不实现),即便 `apiFormat=responses` 也强制走本地 ResponsesAdapter。

frontend:协议类型 UI 改 `label + readonly form-control`(替代误导性 toggle)+「自定义第三方」preset 卡片(`bi-puzzle`,无限重复添加)+ Responses 协议解锁 default 模型映射可空(direct 模式不需 alias)+ baseUrl `autocapitalize="off"` 防 macOS 智能纠错 + 名称用 placeholder 不预填 value + form-submit direct 模式必填 baseUrl/apiKey 校验。

### 2. 诊断闭环(测速 401/403 分级 + 上游错误处理)

- **测速 401/403 改绿色 + 文案明示**:`reachable=true` + 新 `authStatus="auth_required_or_invalid"` + **绿色显示** + 文案 `connection OK; API key not configured or auth not verified`,解耦"连接性"和"鉴权"语义(避免黄色让用户误以为 baseUrl 错)。真鉴权失败由 Codex CLI 实际请求路径暴露
- **`forward.rs` 两处 `unwrap_or_default()`**(web_search retry 检测 + retry 后 4xx/5xx)改 `match` + telemetry.logs.add WARN,避免 body read 失败时静默吞错丢失 root cause
- **`provider_test_error_label` 拆 7 类**(Timeout / ConnectError / RedirectError / DecodeError / RequestError / BodyError / OtherError),用户 toast self-debug
- **`one_million_catalog_ready` catalog 读取/解析失败**改用 `proxy_telemetry().logs.add` 写日志面板可见(原 `.ok().and_then().unwrap_or_else()` 把"文件不存在"和"JSON 损坏"混吃成空对象)

### 3. 全局 `tracing` → `proxy_telemetry().logs` 桥接(silent failure 根治)

workspace 5 处历史 `tracing::warn!`(`registry/healing.rs::heal_builtin_provider_fields` apiFormat 强制覆盖 + `adapters/lib.rs::warn_once_drop_tool` / `disable_web_search_for` 等)在 Tauri 桌面 binary 里**完全静默**(没注册 `tracing_subscriber`,event 默认 drop)。

新加 `TelemetryLogsLayer`(`src-tauri/src/telemetry_bridge.rs`)实现 tracing_subscriber Layer trait,`main()` 第一行 init 全局 subscriber,把所有 `tracing::*` event 转发到 `proxy_telemetry().logs` 同通道(`~/.codex-app-transfer/logs/proxy-*.log` + 设置面板 logs viewer 双可见):

- **`LevelFilter::INFO` 兜底**防未来 dep 引入 TRACE 级 per-byte event 淹没 logs viewer
- **`StringVisitor` sensitive field redact**:`api_key` / `authorization` / `bearer` / `token` / `secret` / `password` 含子串 case-insensitive → `[REDACTED]`,防未来调用方误把 secret 放 tracing field 直接进 logs viewer
- **`try_init` 失败兜底**调 `proxy_telemetry().logs.add("ERROR", ...)`;成功 emit INFO `tracing-bridge active` 正向确认

未来任何 crate 加 `tracing::*` 调用自动生效。

## English

### 1. Custom Third-Party + Responses protocol direct passthrough (main theme)

When users explicitly pick `apiFormat=responses` on the new "Custom Third-Party" preset card and provide both baseUrl + apiKey, **Codex.app connects directly to upstream, bypassing the proxy** (codex-account-switch style pure-config switch): `~/.codex/config.toml::openai_base_url` ← user baseUrl, `auth.json` ← user apiKey, local 18080 port not started. Suitable for OpenAI official / any reverse proxy implementing OpenAI Responses API natively.

**v1.x MiMo Token Plan 404 lesson preserved**: healing on startup force-overrides 8 builtin presets' `apiFormat` back to `openai_chat`; bypass only triggers on **explicit custom + user-chosen** third-party providers — informed choice path only.

Backend `ResponsesPassthroughAdapter`: byte-level passthrough; SSE envelope / `sequence_number` / resp_id all produced by upstream. **Exception**: `/responses/compact` (private extension) always stays on local ResponsesAdapter to avoid upstream 404.

Frontend: protocol type UI now `label + readonly form-control` (replacing the misleading toggle) + "Custom Third-Party" preset card (`bi-puzzle`, addable unlimited times) + Responses unlocks default model mapping as optional + baseUrl `autocapitalize="off"` + name uses placeholder + form-submit direct-mode requires both baseUrl + apiKey.

### 2. Diagnostic loop (speed test message tiers + upstream error handling)

- Speed test 401/403: `reachable=true` + new `authStatus="auth_required_or_invalid"` + **green** display + message `connection OK; API key not configured or auth not verified`, decoupling "connectivity" from "auth" semantics. Real auth failures surface via Codex CLI's actual request path
- `forward.rs` two `unwrap_or_default()` paths replaced by `match` + telemetry.logs.add WARN
- `provider_test_error_label` split into 7 categories
- `one_million_catalog_ready` catalog read/parse failures → `proxy_telemetry().logs.add` (visible in logs viewer)

### 3. Global `tracing` → `proxy_telemetry().logs` bridge (silent-failure root cure)

5 historical `tracing::warn!` call sites across the workspace (healing apiFormat force-override / `warn_once_drop_tool` / `disable_web_search_for` etc.) were **completely silent in the Tauri desktop binary** (no `tracing_subscriber` registered).

New `TelemetryLogsLayer` (`src-tauri/src/telemetry_bridge.rs`) implementing `tracing_subscriber::Layer`, registered at `main()` first line; bridges all `tracing::*` events into `proxy_telemetry().logs` same channel as forward.rs:

- `LevelFilter::INFO` cap to prevent future dep TRACE noise
- Sensitive-field redaction (`api_key` / `authorization` / `bearer` / `token` / `secret` / `password` substring → `[REDACTED]`)
- `try_init` failure → ERROR to telemetry; success → INFO `tracing-bridge active`

Any future crate's `tracing::*` calls auto-active.
