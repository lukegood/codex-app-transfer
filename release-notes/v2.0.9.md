## 🔥 Critical fix: long-session auto-compact stuck on third-party providers

**Symptom**: Long sessions on MiMo / Kimi / DeepSeek / MiniMax / Zhipu / Bailian fail with `404 /responses/compact` when Codex CLI tries to auto-compact.

**Root cause**: `/responses/compact` is a private OpenAI Responses-API endpoint, implemented only by OpenAI. Gateways like litellm only forward it for the `openai` provider; third parties throw `COMPACT not supported`.

**Fix (three parts)**:

(1) **Implement `/responses/compact` locally** (`compact.rs`): route to ResponsesAdapter, repack `CompactionInput` as a chat-completions summarize request, inject Codex's own `SUMMARIZATION_PROMPT`, then wrap the upstream summary back into `{output:[{type:"compaction",encrypted_content:"<SUMMARY_PREFIX>\n<text>"}]}`.

(2) **Write `auto_compact_token_limit` into the catalog**: the missing field used to fall back to `i64::MAX`, so triggering never happened even at 90%. `model_to_json` now writes `context_window × 75%` for every model, leaving 25% buffer for the summarize call itself.

(3) **Cross-turn context inheritance + thinking-provider reasoning passthrough** (PR #53): the CLI puts the compaction item into history, but `input_item_to_messages` didn't recognize `type:"compaction"` and dropped it → amnesia. Fix: recognize `compaction` / `context_compaction` / `compaction_summary` aliases and inject `encrypted_content` as a user message (matching Codex inline compact). Also pass through original `reasoning` + `tools` fields so the thinking-provider placeholder logic fires (avoids Kimi/DeepSeek 400).

Zero changes to Codex CLI config; resume picker unchanged; **all existing sessions remain visible**.

| Provider | v2.0.8 | v2.0.9 |
|---|---|---|
| MiMo / Kimi / DeepSeek / MiniMax / Zhipu / Bailian | ❌ never triggers, or 400 in thinking mode | ✓ triggers at 75% + local summarize + no amnesia |
| OpenAI official | ✓ native | ✓ unchanged |

## Mainline fixes

**(1) MiniMax M2.x compatibility** (PR #47, thanks [@lukegood](https://github.com/lukegood) — first external contributor): MiniMax returned `2013 invalid chat setting` because it rejects the full OpenAI parameter set. `sanitize_minimax_chat_body` (gated on minimax/minimaxi only) retains a field whitelist, drops `reasoning_effort`/`parallel_tool_calls`, forces `reasoning_split=true`, merges consecutive system, strips tool `strict`, downgrades `tool_choice`. Response side: parse `reasoning_details[].text` + `<think>` state-machine fallback (buffers tags split across chunks). **Gated MiniMax-only** so DeepSeek/Kimi literal `<think>` text isn't misread as reasoning.

**(3) Arbitrary `context_window` values** (PR #45): old code picked between `258_400` / `1_000_000` only (`mimo-v2.5-pro` is actually 1M but was tagged 258_400, causing early compaction). `model_capabilities[<model>].context_window` now wins when ≥ 1024; the two-bucket default is only fallback.
