**修法**:基于 [Piebald-AI/claude-code-system-prompts](https://github.com/Piebald-AI/claude-code-system-prompts) 反编译公开版本,把 prompt 改写成 Claude Code 风格 9-section 强 schema:

- 强制 `<analysis>` + `<summary>` 二段输出(chain-of-thought 先做时序分析再生 summary)
- 9 固定 section:`Primary Request and Intent` / `Key Technical Concepts` / `Files and Code Sections`(完整 path + snippets)/ `Errors and Fixes` / `Problem Solving` / `All User Messages`(逐字列防丢主诉)/ `Pending Tasks` / `Current Work` / `Next Step`(verbatim quote 最近用户诉求)

- 注入位置从 system 改成最后一条 user message(对齐 Codex CLI 自家做法,提升第三方 provider 服从度)
- 响应解析只抽 `<summary>` 段进续轮 history,过滤 `<analysis>` chain-of-thought 防污染续轮模型注意力

| Compact 体感 | v2.0.x | v2.1.0 |
|---|---|---|
| 历次 user 主诉 | ❌ 只记最后 1-2 条 | ✓ ALL User Messages 段逐字列 |
| 当前任务目标 | ❌ 经常丢 | ✓ Next Step verbatim quote 最近 user |
| 文件路径 / 命令字面量 | ❌ 模糊"已检查相关文件" | ✓ Files and Code Sections 完整 path + snippets |

## English

## 🔥 Key changes: macOS Intel x64 support + dramatic auto-compact summary upgrade

First minor release in the v2.x line. Two main themes: **new platform support (macOS Intel)** and **auto-compact summary quality now approaches Claude Code's experience**.

### 1. macOS Intel x64 build matrix (closes [#61](https://github.com/Cmochance/codex-app-transfer/issues/61))

`release.yml` now includes `macos-15-intel + x86_64-apple-darwin + macOS-x64` for native Intel builds (`macos-15-intel` is GitHub's new runner label introduced after macos-13's retirement to extend Intel x64 support — free for public repos, officially supported through 2027-08). All macOS-specific steps (Apple signing, notarization, ad-hoc fallback, codesign verify) use `startsWith(matrix.os, 'macos-')` so they cover both ARM and Intel runners. `xtask release_bundle.rs` already had the `-macOS-x64.dmg` regex pattern reserved, so it auto-recognizes `macos-x64` as a platform key in `latest.json`. Intel Mac users can now download `Codex-App-Transfer-v2.1.0-macOS-x64.dmg` directly from the Releases page — matching OpenAI Codex's "separate arm64 / Intel artifacts" style, zero size waste (arm64 users don't pull Intel binaries).

### 2. Auto-compact summary now approaches Claude Code's quality (fixes the "amnesia" experience)

**Symptom** (user feedback): the v2.0.x local `/responses/compact` implementation used Codex CLI's 86-character SUMMARIZATION_PROMPT. With third-party providers (Kimi/MiMo/DeepSeek) — whose instruction-following is weaker than GPT-5 — the model freely "topic-summarized" history and lost user requests / file paths / command literals → "I'm curling a webpage" but couldn't recall *which* page or *why*.

**Fix**: rewrote the prompt based on [Piebald-AI/claude-code-system-prompts](https://github.com/Piebald-AI/claude-code-system-prompts) reverse-engineered public version, into a Claude-Code-style 9-section strict schema:

- Two-pass output: `<analysis>` (chronological CoT) → `<summary>` (final structured report)
- Nine fixed sections: `Primary Request and Intent` / `Key Technical Concepts` / `Files and Code Sections` (full paths + verbatim snippets) / `Errors and Fixes` / `Problem Solving` / `All User Messages` (listed verbatim — preserves intent shifts) / `Pending Tasks` / `Current Work` / `Next Step` (with verbatim quote of the most recent user message)
- 30-line few-shot example at the end — prevents third-party providers' output format drift
- Prompt is now injected as the **last user message** instead of system, aligning with Codex CLI's own behavior and improving third-party compliance
- Response parser extracts only `<summary>` into the resumed history, filtering out `<analysis>` chain-of-thought to avoid polluting the next turn

| Compact experience | v2.0.x | v2.1.0 |
|---|---|---|
| Earlier user requests | ❌ Only last 1-2 remembered | ✓ "All User Messages" section lists them verbatim |
| Current task goal | ❌ Frequently lost | ✓ "Next Step" includes a verbatim quote of the latest user message |
| File paths / command literals | ❌ Vague "checked relevant files" | ✓ "Files and Code Sections" with absolute paths + verbatim snippets |
| Next-turn model attention | ⚠️ Chain-of-thought leaked into history | ✓ Only `<summary>` is preserved; `<analysis>` is stripped |
